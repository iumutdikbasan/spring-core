Dependency Injection (DI)

Sınıflar birbirlerini kullanarak yapılacak olan işlemleri gerçekleştirirler. Bu sınıflar arası bağımlılıkların oluşmasını sağlar.


Örnek

public class UserController {
    private UserServiceImpl userServiceImpl = new UserServiceImpl();
}

Yukarıdaki örnekte UserController sınıfı belirli işlemleri gerçekleştirebilmek için UserServiceImpl sınıfına bağımlıdır. Fakat UserController sınıfı içerisinde new operatörü ile nesne oluşturulması,
UserController sınıfını direkt olarak UserServiceImpl sınıfına bağımlı kılmaktadır. Bu Dependency Inversion Principle’a aykırıdır. Çünkü bağımlılığın yönünün soyut sınıflara doğru olması gerekirken burada somut bir sınıfa doğru olmuştur.
Herhangi bir şekilde UserServiceImpl sınıfı değişikliğe uğrarsa, UserController bu değişiklikten direkt olarak etkilenecektir.


    public class UserController {
    private UserService userService;
    public UserController(UserService userService) {
        this.userService = userService;
    }
}

Yukarıdaki örnekte ise UserController sınıfı bu sefer somut bir sınıfa bağımlı olmamakla birlikte new operatörünü kullanarak bağımlılığını kendisi oluşturmuyor.
Peki bu bağımlılık nasıl oluşuyor? UserController sınıfı oluşturulduğu anda constructor metoduna uygulama çatısı tarafından herhangi bir UserService implementasyonu gönderilerek bu bağımlılık enjekte ediliyor.
Bu işleme Dependency Injection yani bağımlılıların enjekte edilmesi denir.
